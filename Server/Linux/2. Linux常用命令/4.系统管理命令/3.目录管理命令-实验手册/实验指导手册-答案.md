#### 3.1、实验原理

##### man
- 使用man这个命令可以调阅其中的帮助信息，非常方便和实用。在输入命令有困难时可以立即查阅相关帮助信息。
- 语法:

```shell
man [option] command

option：man命令的选项

command：所需要查询帮助的命令
```


##### history
- 当前终端命令行输入并执行命令的时，Linux会自动把命令记录到历史列表中，一般保存在用户HOME目录下的.bash_history文件中。默认保存1000条，这个值是可以更改的。如果不需要查看历史命令中所有项目，history可以只查看最近n条记录列表。history命令不仅可以查询历史命令，而且有相关的功能执行命令。

- 语法:
```shell
histroy [option]

option：history命令的选项
```


##### date
- date命令的功能是显示或设置系统的日期和时间。只有超级用户才能使用date命令设置时间，普通用户只能使用date命令显示时间。

##### uptime
- Linux系统中的uptime命令主要用于获取主机运行时间和查询Linux系统负载等信息。uptime命令可以显示系统已经运行了多长时间，信息显示依次为:现在时间、系统已经运行了多长时间、目前有多少用户登陆、系统在过去的1分钟/5分钟/15分钟内的平均负载。
##### free
- free命令会显示内存的使用情况，包括实体内存、虚拟的交换文件内存、共享内存区段，以及系统核心使用的缓冲区等。
- 语法:
```shell

free [option]
option：free命令的选项

```

##### tree

- 使用tree命令以树状图递归的形式显示各级目录，可以方便地看到目录结构。
- 语法:
```shell
tree [option] directory

option：tree命令的选项

directory：所需解析的目录
```

##### at

- 可以设置在一个指定的时间执行一个指定的任务，只能执行一次，使用前确认系统开启了atd进程。如果指定时间已经过去，则会在第二天执行

- 语法:
```shell
at time command

time: 要执行命令的时间

command: 需要执行的命令
```

##### shutdown
- shutdown命令安全地将系统关机。有些用户会使用直接断掉电源的方式来关闭Linux，这是十分危险的。

- 语法:
```shell
shutdown [OPTION]

OPTION: shutdown命令的选项
```

##### halt

- halt的本质就是调用shutdown -h（关机指令）。halt执行时会杀死相应的进程，执行sync调用，文件系统写操作完成后就停止内核，与root不同之处在于halt用来关机，而reboot用来重启系统。

##### reboot
- reboot命令用于重启系统，用法比较简单，在终端命令行以root用户执行该命令即可进行系统的重启。

- 语法:
```shell
reboot [OPTION]

reboot: shutdown命令的选项
```
##### poweroff
- rpoweroff就是halt或reboot命令的软链接。而执行halt调用shutdown -h，用法就是直接在终端输入poweroff就可以关闭系统。

##### uptime
- uptime用法十分简单，直接输入uptime即可,将所用到的指令和指令执行结果截图

     `uptime`

![[lab1.png]]
- 05:25:06表示当前系统时间，up 4:17表示主机已经运行时间，值越大，说明机器越稳定。2 users表示用户连接数。load average表示系统平均负载，统计最近1分钟、5分钟、15分钟的系统平均负载
- 对于单核CPU来说,负载小于3表示当前系统性能良好；3～10表示需要关注，系统负载可能过大，需要做对应地优化；大于10表示系统性能有严重问题

##### free

- 以M为单位查看系统内存资源占用情况,将所用到的指令和指令执行结果截图

     `free -m`
     
![[lab2.png]]
- Mem:表示物理内存统计。这台机器的物理内存在1000MB
- 808表示系统内从总量为808MB
- 227表示已用内存大小为227M
- 195表示未被分配的内存大小为195MB
- 15表示共享内存为15MB
- 385表示系统分配但未被使用的buffers和cache数量为385MB
- 424表示可用的内存大小为424MB
- Swap:表示硬盘上交换分区的使用情况。如剩余空间较小，需要留意当前系统内存使用情况及负载
- 如果需要计算程序占用内存，可以使用一下公式计算: total - free -buff/cache - shared=808 - 227 - 385 - 15 = 181，内存使用百分比为181/808=22%
