# 课题一：服务器与客户端的区别（Windows Server 2016 场景）

## 教学目标
- 理解“服务器/客户端”的角色与职责差异
- 识别硬件、系统、网络、可靠性、授权等方面的不同
- 通过现场演示体会“服务器 = 提供服务的系统角色”
## 知识点与讲授顺序

1. **角色定位**
- 服务器（Server）：持续运行、对外提供服务（Web/AD/DNS/DB/文件共享等），强调稳定、可扩展、可维护。
- 客户端（Client/Workstation）：面向个人或办公终端，强调交互体验与多媒体。
2. **硬件层面**
- 服务器常见：多路/多核CPU、ECC内存、RAID/企业级SSD、冗余电源、IPMI/iLO/iDRAC 远程管理。
- 客户端常见：单路CPU、非ECC内存、单电源，侧重显卡/外设体验。
3. **操作系统特性**
- Windows Server 2016 提供“角色与功能”（AD DS、DNS、DHCP、IIS、文件服务、Hyper-V 等）。
- 客户端（如 Windows 10）：缺少或简化服务端角色，面向本地应用与办公。
4. **网络与并发能力**
- 服务器支持大量并发连接、长时间监听端口、集中式身份/策略管理。
- 客户端通常作为发起方，偶尔共享资源，连接数与策略管理能力有限。
5. **可靠性与维护**
- 服务器强调：补丁窗口、变更控制、监控告警、备份与容灾（RAID、快照、异地容灾）。
- 客户端强调：个人数据保护与简单还原。
6. **授权与成本**
- 服务器版授权（版本与CAL/核心授权等）与客户端零售/OEM授权模式不同。
- TCO：硬件+授权+机房/云资源+运维人力。

# 课题二：操作系统种类概览（Windows / Linux）

## 教学目标
- 掌握 OS 基本构成与通用概念（内核、用户态、进程、文件系统、服务/守护进程）
- 了解 Windows 与 Linux 的主流分支、典型管理方式与应用场景
- 能在命令行快速辨识系统版本，并说出常见包管理与服务管理方法
    
## 知识点与讲授顺序

1. **操作系统通用概念**
- 内核（驱动/进程调度/内存/文件系统/网络栈）+ 用户态（应用/命令行/GUI）。
- 服务（Windows Service / Linux Daemon）、启动与日志、包/更新机制。
2. **Windows 家族**
- **桌面**：Windows 10/11（面向交互）。
- **服务器**：Windows Server 2016（Essentials/Standard/Datacenter；Desktop Experience vs Server Core）。
- **管理方式**：GUI（Server Manager）、PowerShell、组策略、远程管理（MMC/WinRM/RDP）。
- **典型场景**：AD 域、文件与打印、IIS、.NET 应用、Hyper-V、RDS 等。
3. **Linux 家族**
- **发行版谱系**：
    - RHEL 系（RHEL、AlmaLinux、Rocky Linux；包管理 `yum/dnf`），企业稳定、文档完善。
    - Debian/Ubuntu 系（包管理 `apt`），社区活跃、云原生常见。
    - SUSE/OpenSUSE（包管理 `zypper`），在特定行业与企业环境常见。
- **管理方式**：SSH、Shell、包管理、`systemd` 服务、日志（`journalctl`/`/var/log`）。
- **典型场景**：Web/Nginx、容器与微服务、数据库、中间件、自动化/DevOps。
