# 《主流 Web 开发语言与 Web 服务器》教案  
（以 Nginx / Apache / Python / PHP / Java 为教学案例）

---

### 一、课程内容
认识主流 Web 开发语言（前端与后端）及常见 Web 服务器（Nginx、Apache、Caddy、IIS 等），理解现代 Web 架构与语言生态的特点及发展趋势。  

![[05.部署FTP服务_基础答案.docx]]

### 二、课时安排
共 2 课时（理论讲授 + 案例演示）  

---

### 三、教学方式
讲授 + 案例分析 + 对比演示 + 讨论互动  

---

### 四、教学目标
#### （一）知识目标
- 理解 Web 前端与后端的基本分工与协作关系；  
- 掌握主流 Web 开发语言（HTML、CSS、JavaScript、PHP、Python、Java、Go 等）的特点与应用场景；  
- 熟悉主流 Web 服务器（Nginx、Apache、Caddy、LiteSpeed、IIS）的功能与适用环境；  
- 理解现代 Web 架构向开源、轻量化、跨平台发展的趋势。  

#### （二）能力目标
- 能区分前端与后端语言的作用与职责；  
- 能说明不同 Web 服务器的适用场景与配置差异；  
- 能在实验环境中搭建最基本的 Web 服务（如 Nginx + PHP / Flask / Spring Boot）。  

#### （三）素质目标（含思政元素）
- 通过学习开源 Web 技术，树立自主创新与合作共享的意识；  
- 理解开源生态对我国信创产业和互联网发展的重要意义；  
- 培养“技术报国”的使命感，激发学生探索新一代 Web 技术（如云原生、AI 服务）的热情。  

---

### 五、教学重点
- 前端与后端语言的功能定位与代表框架；  
- 主流 Web 服务器的特性、差异与趋势；  
- Web 架构从单体到云原生的演进方向。  

---

### 六、教学难点
- 理解多语言协同（前端 + 后端）的工作机制；  
- 理解 Web 服务器与应用框架之间的关系。  

---

### 七、教学方法
讲授法、演示法、案例分析法、对比讨论法  

---

### 八、教学过程

#### （一）导入环节（约10分钟）
1. 教师提问：“一个网页从你点击访问到显示出来，背后发生了什么？”  
2. 引导学生思考 Web 服务端与浏览器之间的通信；  
3. 引出本课主题——Web 开发语言与服务器的协同关系。  

---

#### （二）讲解示范环节（约50分钟）

##### 1. 前端开发语言
- **HTML**：负责网页结构，定义页面框架。  
- **CSS**：负责样式与布局，让页面更美观。  
- **JavaScript**：负责交互与逻辑，实现动态效果。  
- **TypeScript**：JavaScript 的增强版，支持类型检查，适合大型前端项目。  

> **演示建议**：  
> 在浏览器中打开一个 HTML 文件，使用 JavaScript 修改页面内容（如动态显示当前时间）。  

---

##### 2. 后端开发语言
- **PHP**：简单易学，广泛应用于 CMS 与中小网站（WordPress、Discuz）。  
- **Python**：语法简洁，框架丰富（Flask、Django、FastAPI），适合数据驱动与AI服务。  
- **Java**：稳定可靠，企业级项目首选（Spring、Spring Boot）。  
- **C# / ASP.NET**：微软生态语言，结合 IIS 或 Kestrel。  
- **Node.js (JavaScript)**：前后端统一语言，适合实时通信与高并发。  
- **Go (Golang)**：高性能、简洁语法，适合API与微服务。  
- **Ruby / Rails**：快速开发原型系统的优秀语言。  

> **对比说明**：  
> - PHP 与 Python 都适合中小型应用；  
> - Java 与 Go 更适合大型分布式系统；  
> - Node.js 在实时性与事件驱动场景下优势明显。  

---

##### 3. 主流 Web 服务器
| 服务器 | 特点 | 适用场景 |
|---------|------|-----------|
| **Nginx** | 高性能、支持反向代理与负载均衡、跨平台 | 企业主流选择 |
| **Apache HTTPD** | 历史悠久、模块丰富、兼容性好 | 传统项目、CMS系统 |
| **Caddy** | 自动HTTPS、配置简洁 | 现代开发环境、个人项目 |
| **LiteSpeed / OpenLiteSpeed** | 高性能PHP环境、商用化 | WordPress与虚拟主机 |
| **IIS** | 仅限Windows Server、管理直观 | 旧版ASP.NET项目 |

> **演示建议**：  
> 在课堂中展示 Nginx 的简单配置文件：  
> ```nginx
> server {
>     listen 80;
>     server_name localhost;
>     root /var/www/html;
>     index index.html index.php;
> }
> ```  
> 启动服务后在浏览器中访问 `http://localhost` 观察页面。  

---

##### 4. 为什么 IIS 不再是主流
- 只能运行在 Windows Server 平台，缺乏跨平台性；  
- 开源生态不活跃，资料少、扩展弱；  
- 现代 .NET 项目已转向 Kestrel + Nginx / Caddy 反代模式；  
- 云原生趋势推动轻量化、容器化架构，Nginx / Caddy 更适应。  

> **思政延伸**：  
> 我国的信创生态正朝着开源、自主可控方向发展，Web 架构的开放性与兼容性正体现着信息主权的重要性。  

---

#### （三）任务实施环节（约20分钟）
1. 学生分组任务：  
   - 组A：搭建最小化 Nginx + PHP 环境，显示 `index.php` 页面；  
   - 组B：使用 Python Flask 创建简单 Web 服务（返回“Hello Web!”）；  
   - 组C：比较访问延迟与配置差异。  
2. 汇报：每组总结一种语言 + 服务器的特点。  

---

#### （四）分析总结环节（约10分钟）
- 前端与后端协作构成完整 Web 应用；  
- 不同语言与服务器对应不同业务场景；  
- Web 技术的发展体现了开放协作与创新精神。  

---

### 九、课后拓展
- 阅读资料：《Nginx 官方文档》与《Flask 框架入门指南》；  
- 调研题：目前主流互联网公司使用的后端语言与Web框架；  
- 实践题：在虚拟机或 Docker 中部署 Nginx + Flask 小型网站。  

---

### 十、教学反思
- 学生对前后端协作模式兴趣浓厚，但需进一步理解HTTP请求全过程；  
- 建议在后续课程中结合动态网页部署实验；  
- 思政引导良好，学生能理解“技术创新、自主可控”的国家战略方向。
