# 主流 Web 开发语言
## Web 开发语言主要分为 **前端语言** 和 **后端语言**。

### 1. 前端开发语言
前端负责用户界面和交互：
- **HTML**：网页结构的标记语言。
- **CSS**：控制页面样式和布局。
- **JavaScript**：让网页具备交互能力，可操作 DOM、与后端交互（Ajax/Fetch）。
- **TypeScript**：JavaScript 的超集，支持静态类型检查，前端大型项目常用。
### 2. 后端开发语言
后端负责逻辑处理、数据库交互、用户认证：
- **PHP**
    - 简单易学，配合 Apache/Nginx 使用广泛。
    - 常见框架：Laravel、ThinkPHP。
    - 应用：WordPress、Discuz、Drupal 等。
- **Python**
    - 简洁易用，生态丰富。
    - 常见框架：Django、Flask、FastAPI。
    - 应用：数据处理、AI 与 Web 结合的服务。
- **Java**
    - 企业级 Web 开发的常青树。
    - 常见框架：Spring、Spring Boot。
    - 应用：大型企业应用（电商、金融）。
- **C# / ASP.NET**
    - 微软生态下的 Web 语言。    
    - 与 IIS 搭配良好，适合 Windows 环境。    
- **JavaScript (Node.js)**
    - JavaScript 在后端的扩展。
    - 特点：单线程、事件驱动，适合高并发 IO。
    - 应用：实时通信（如聊天系统）。   
- **Go (Golang)**
    - 谷歌开发，性能接近 C，开发效率高。    
    - 特点：适合构建高并发的 API 服务。    
    - 常见框架：Gin、Beego。    
- **Ruby**
    - 以简洁优雅著称。    
    - 常见框架：Ruby on Rails。    
    - 应用：快速开发原型网站。


## 主流 Web 服务器
**主流 Web 服务器** 应该是：
- **Nginx**（事实上的标准，Web/反代/负载均衡全能）
- **Apache HTTPD**（老牌，仍在很多老项目和 CMS 中广泛使用）
- **Caddy**（新兴，配置极简，自带 HTTPS，越来越受开发者欢迎）
- **LiteSpeed/OpenLiteSpeed**（高性能商用，特别是 PHP 环境）
- （某些特定场景）**Envoy、HAProxy**（更偏向服务网格和负载均衡）

## IIS不再是主流服务器的原因
- **跨平台劣势**：只能跑在 Windows Server 上，而现代 Web 服务更多选择 Linux 生态（Nginx、Apache、Caddy、Envoy 等）。
- **社区生态弱**：相比开源的 Nginx、Apache，IIS 的社区活跃度和资料积累要差很多。
- **现代架构转移**：现在 .NET Core / .NET 6+ 都支持跨平台运行，很多新项目直接部署在 Kestrel（内置 Web 服务器）+ 反向代理（Nginx、Caddy）上，而不是依赖 IIS。
- **云原生趋势**：容器化、Kubernetes 普及，使得轻量化、开源、跨平台的 Web 服务器成为主流。
