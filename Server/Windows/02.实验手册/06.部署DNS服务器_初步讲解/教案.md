# 《DNS服务器的部署与解析》教案  
（以 Windows Server 2016 为教学案例）

---

### 一、课程内容
认识 DNS（域名系统）的作用与结构，掌握在 Windows Server 2016 上部署 DNS 服务器、创建正向与反向查找区域及主机记录的全过程，理解 DNS 域名解析机制，并能通过命令行工具验证解析结果。

---

### 二、课时安排
共 2 课时（理论讲授 + 实验操作）

---

### 三、教学方式
讲授 + 演示 + 任务驱动 + 小组实验  

---

### 四、教学目标
#### （一）知识目标
- 理解 DNS 的基本概念、作用与工作原理；  
- 掌握 DNS 服务器角色的安装步骤；  
- 掌握创建正向查找区域与反向查找区域的方法；  
- 理解 A 记录、CNAME 记录、PTR 记录的作用与配置方式。  

#### （二）能力目标
- 能够独立部署 DNS 服务器；  
- 能在 DNS 管理器中创建区域与主机记录；  
- 能通过 `nslookup` 验证正向与反向解析结果；  
- 能结合 IIS 服务完成域名访问测试。  

#### （三）素质目标（含思政元素）
- 通过 DNS 实验，培养学生系统思维与网络服务协同意识；  
- 理解互联网命名系统的有序与统一，体会“规范与规则”在信息社会中的重要性；  
- 树立“安全运行、科学规划、服务社会”的职业责任感。  

---

### 五、教学重点
- DNS 正向查找区域与反向查找区域的配置；  
- 主机记录（A）、别名记录（CNAME）与反向记录（PTR）的创建与验证。  

---

### 六、教学难点
- 理解 DNS 层级结构与名称解析过程；  
- 掌握正反向解析对应关系及其测试方法。  

---

### 七、教学方法
讲授法、演示法、任务驱动法、实验验证法  

---

### 八、教学过程

#### （一）导入环节（约10分钟）
1. 提问：“当你在浏览器输入 www.baidu.com 时，系统是如何找到服务器的？”  
2. 引导学生回顾 IP 地址与主机名的关系，引出 DNS 的作用。  
3. 说明本节课目标：搭建一套完整的 DNS 实验环境，实现从域名到 IP 的自动解析。  

---

#### （二）讲解示范环节（约50分钟）

##### 1. 实验拓扑与环境配置
- 在 VMware Workstation 中准备三台虚拟机：

| 机器名 | 系统版本 | IP | 作用 |
|---------|-----------|-----------|--------|
| iLgj-Server-xx-01 | Win2016 | 192.168.xx.10 | DNS 服务器 |
| iLgj-Server-xx-02 | Win2016 | 192.168.xx.11 | Web 服务器 |
| iLgj-Server-xx-03 | Win2016 | 192.168.xx.12 | 客户端 |

- 网络：同网段直连；网关与DNS均指向 192.168.xx.10。  

---

##### 2. 部署 Web 服务（实验准备）
1. 在 iLgj-Server-xx-02 上安装 IIS Web 服务器角色；  
2. 在 `C:\WebRoot` 下创建网站根目录与主页 `index.html`，写入内容“DNS IIS xx”；  
3. 在 IIS 管理器中创建网站：  
   - 站点名称：`Site`  
   - 路径：`C:\WebRoot`  
   - 默认端口：80  
   - 验证页面能通过 `http://192.168.xx.11` 正常访问。  

---

##### 3. 部署 DNS 服务器（核心部分）
1. 在 iLgj-Server-xx-01 上安装 DNS 服务器角色；  
2. 打开 DNS 管理器，新建正向查找区域：  
   - 区域名称：`testxx.com`  
   - 区域类型：主要区域（Primary Zone）；  
   - 允许动态更新：否。  
3. 在 `testxx.com` 区域中添加主机记录（A 记录）：  
   - 名称：`www`  
   - IP 地址：`192.168.xx.11`（指向 Web 服务器）。  
4. 添加别名记录（CNAME）：  
   - 名称：`www1`  
   - 指向：`www.testxx.com`。  
5. 新建反向查找区域：  
   - 网络 ID：`192.168.xx`；  
   - 区域类型：主要区域。  
6. 在反向区域中添加 PTR 记录：  
   - IP 地址：`192.168.xx.11`  
   - 指向：`www.testxx.com`。  

> **演示重点**  
> - 区域名称与主机记录的层级关系；  
> - A、CNAME、PTR 三类记录的作用区别；  
> - 正向与反向解析路径。  

---

##### 4. 解析验证
1. 在客户端 iLgj-Server-xx-03 上配置 DNS 地址为 192.168.xx.10；  
2. 打开命令提示符执行：  
   - `nslookup www.testxx.com` → 应返回 IP 地址 192.168.xx.11；  
   - `nslookup 192.168.xx.11` → 应返回主机名 www.testxx.com；  
3. 打开浏览器访问：  
   - `http://www1.testxx.com`  
   - 页面应正常显示“DNS IIS xx”。  

> **讲解要点**  
> - 正向解析：域名 → IP；  
> - 反向解析：IP → 域名；  
> - 若无法解析，检查 DNS 地址、记录类型或缓存刷新（`ipconfig /flushdns`）。  

---

#### （三）任务实施环节（约20分钟）
1. 学生分组完成以下任务：  
   - 安装并配置 DNS 服务器角色；  
   - 创建 testxx.com 区域及 A、CNAME、PTR 记录；  
   - 使用 `nslookup` 验证正反解析；  
   - 打开浏览器访问别名站点验证结果。  
2. 小组提交截图与实验报告。  
3. 教师巡视指导与讲评。  

---

#### （四）分析总结环节（约10分钟）
- DNS 是网络通信的“通讯录”，实现了域名与 IP 的映射；  
- 正向区域实现“名称找地址”，反向区域实现“地址找名称”；  
- 实际工作中，DNS 常用于企业内部系统访问、AD 域环境、负载均衡与冗余部署。  

---

### 九、课后拓展
- 阅读资料：《Windows Server DNS 管理指南》《BIND 9 概念手册》；  
- 思考题：  
  1. DNS 正向与反向查找区域有何区别？  
  2. 主机记录在 DNS 解析中扮演什么角色？  
  3. 企业级 DNS 还能实现哪些高级功能？（如转发、辅助区、根提示）  
- 实践延伸：  
  - 使用 PowerShell 创建 DNS 区域与记录；  
  - 在 Linux 上使用 `dig` 命令验证跨平台解析。  

---

### 十、教学反思
- 学生在实验中易混淆“正向区名”与“主机记录名”，可通过示意图强化理解；  
- DNS 实验与 Web 服务结合性强，效果直观，可作为网络服务综合项目；  
- 思政点融入自然，学生能体会“规范命名、系统协同、服务社会”的价值理念。
