# 《Windows 用户与权限管理》教案  
（以 Windows Server 2016 为教学案例）

---

### 一、课程内容
认识 Windows Server 用户账户、用户组及 NTFS 与共享权限的配置与关系。  

---

### 二、课时安排
共 2 课时（理论 + 实践演示）  

---

### 三、教学方式
讲授 + 案例演示 + 实验操作 + 任务驱动  

---

### 四、教学目标
#### （一）知识目标
- 理解 Windows 用户账户与组的概念及其关系；  
- 掌握 NTFS 权限与共享权限的作用对象、配置方法及生效逻辑；  
- 理解“权限合并”与“拒绝优先”的原则。  

#### （二）能力目标
- 能在 Windows Server 2016 中创建用户、用户组并分配权限；  
- 能通过实践验证 NTFS 与共享权限的交集规则；  
- 能正确判断用户最终有效权限。  

#### （三）素质目标（含思政元素）
- 通过账户与权限管理的学习，理解“职责分明、授权有度”的组织理念；  
- 树立安全意识与团队协作精神，理解制度化管理在信息安全中的重要性；  
- 培养“严谨、规范、可信”的职业品质，助力构建网络安全防线。  

---

### 五、教学重点
- 用户与组的关系与权限继承原理  
- NTFS 与共享权限的交互机制及有效权限判定  

---

### 六、教学难点
- 权限叠加与拒绝优先原则的理解与验证  

---

### 七、教学方法
讲授法、演示法、任务驱动法、比较分析法  

---

### 八、教学过程

#### （一）导入环节（约10分钟）
1. 教师提问：“如果每个员工都能访问服务器的所有文件，会出现什么问题？”  
2. 通过引导学生思考企业内部信息安全与权限管理的重要性，引出本节主题。  
3. 说明本课将围绕“账户与组管理”及“NTFS与共享权限控制”两部分展开。  

---

#### （二）讲解示范环节（约50分钟）

##### 1. 用户与组管理
- **用户账户的概念**  
  - 用户 = 身份 + 密码 + 权限。  
  - 每个用户拥有唯一的安全标识符（SID）。  
  - 默认账户：Administrator、Guest。  

- **组的概念**  
  - 组 = 权限集合，用于方便批量管理。  
  - 默认组：Administrators、Users、Remote Desktop Users 等。  

- **用户与组的关系**  
  - 用户可以属于多个组。  
  - 有效权限 = 用户权限 ∪ 所属组权限。  

> **演示1：创建用户与组**
> - 在 Windows Server 2016 上创建用户 `student1`，设置密码。  
> - 创建组 `ProjectTeam`，并将 `student1` 加入该组。  
> - 用 `student1` 登录系统，验证权限。  

---

##### 2. NTFS 与共享权限
- **NTFS 权限**  
  - 作用于 NTFS 分区的文件/文件夹。  
  - 常见权限：完全控制、修改、读取和执行、列出内容、读取、写入。  
  - 可继承：子文件夹可继承父目录权限。  

- **共享权限**  
  - 作用于网络访问时的共享资源。  
  - 常见权限：读取、更改、完全控制。  
  - 仅在“网络访问”时生效。  

- **两者关系**
  - 有效权限 = 共享权限 ∩ NTFS 权限（取交集，最严格者生效）。  

- **权限合并原则**
  - 允许权限叠加（累加）。  
  - 拒绝权限优先，覆盖所有允许权限。  

> **演示2：权限验证实验**
> 1. 创建文件夹 `D:\ShareTest`；  
> 2. 设置共享 → Everyone: 完全控制；  
> 3. 为用户 `student1` 设置 NTFS 权限为“只读”；  
> 4. 让 `student1` 从网络访问 `\\ServerIP\ShareTest`，验证：只能读，不能写；  
> 5. 修改为“修改”权限后，验证可读写。  

---

#### （三）任务实施环节（约20分钟）
1. 学生分组完成以下任务：  
   - 组A：创建用户 `testA`，分配到组 `DevTeam`；  
   - 组B：创建共享文件夹 `ProjectData`，设置共享权限与NTFS权限；  
   - 组C：验证权限交集与拒绝优先原则。  
2. 各组提交截图与实验结果报告。  
3. 教师巡查并讲评。  

---

#### （四）分析总结环节（约10分钟）
- 权限管理的核心是“最小化授权”；  
- 用户与组的关系体现了**团队分工与协作机制**；  
- NTFS 与共享权限的叠加机制，体现了**安全防护的多层控制思想**；  
- 结合思政点：规范操作、守住权限边界，是信息安全与职业道德的体现。  

---

### 九、课后拓展
- 学习 Active Directory 域环境中的集中账户管理；  
- 了解文件系统权限在 Linux 系统中的实现方式（`chmod`、`chown`）；  
- 思考题：为什么“拒绝优先”是权限系统中必要的设计？  

---

### 十、教学反思
- 学生在实验环节对权限交集与拒绝优先理解较慢，可通过表格总结与对比方式强化；  
- 后续课程可引入 AD 域环境，展示更复杂的权限继承关系；  
- 思政教育融入自然，学生普遍能理解“安全责任”与“规范授权”的职业意义。
