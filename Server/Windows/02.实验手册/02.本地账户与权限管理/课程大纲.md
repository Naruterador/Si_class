# 课题三：用户账户与组（Windows Server 2016）
## 知识点与讲授顺序

1. **用户账户的概念**
- 用户 = 身份 + 密码 + 权限。
- 每个用户有唯一 SID（安全标识符）。
- 默认用户：Administrator、Guest。

2. **组的概念**
- 组 = 权限集合，方便批量分配。
- 默认组：Administrators、Users、Remote Desktop Users 等。

3. **用户与组的关系**
- 用户可以属于多个组。
- 权限继承：用户的有效权限 = 所属组权限的合并。
    
## 演示/实验
- 在 Windows Server 2016 上创建用户 `student1`，并设置密码。
- 创建组 `ProjectTeam`，并将 `student1` 加入该组。
- 登录测试：用 `student1` 登录，查看权限。
    
## 板书要点

- **用户 = 身份**
- **组 = 权限集合**
- **用户可属于多个组**
- **有效权限 = 所属组权限合并**

# 课题四：NTFS 权限与共享权限（Windows Server 2016）
## 知识点与讲授顺序

1. **NTFS 权限**
- 作用对象：存储在 NTFS 分区上的文件/文件夹。
- 常见权限：完全控制、修改、读取和执行、列出文件夹内容、读取、写入。
- 可继承：子文件夹继承父目录权限。
    
2. **共享权限**
- 作用对象：网络共享时的访问控制。
- 常见权限：读取、更改、完全控制。
- 仅在通过“网络访问共享”时生效。
    
3. **两者关系**
- **共享权限** + **NTFS 权限** → 有效权限 = **取交集**（更严格的那一方生效）。
- 例：共享权限“完全控制”，NTFS 权限“只读” → 最终用户只能“只读”。

  4.**权限合并原则**
1. **允许权限是叠加的（累加）**
- 用户属于多个组时，不同组的“允许”权限会合并。
- 举例：
    - 用户本身：只读
    - 组权限：修改
    - 👉 最终有效权限 = 修改（更大）。
2. **显式拒绝优先**
- 如果某个用户或组被设置了“拒绝”某个权限，那么无论其他地方给了多少“允许”，最终都不能获得该权限。
- 👉 拒绝的权重最高。

1. **最佳实践**
- 共享权限设置为 Everyone: 完全控制。
- 实际权限控制交给 NTFS 权限。
    

## 演示/实验

A. **创建共享文件夹**
- 在 D:\ 创建 `ShareTest` 文件夹。
- 设置共享 → Everyone: 完全控制。
    
B. **设置 NTFS 权限**
- 给用户 `student1` 分配“只读”权限。
- 让用户 `student1` 通过网络访问 `\\ServerIP\ShareTest`，验证：可以读，不能写。
    
C. **对比实验**
- 修改 `student1` 的 NTFS 权限为“修改”，再次访问 → 可以读写。
    
## 板书要点
- **NTFS 权限 = 本地磁盘文件/文件夹权限**
- **共享权限 = 网络共享访问权限**
- **有效权限 = 交集（最严格优先）**
- **最佳实践：共享放开，控制靠 NTFS**

## 总结：
- 不是“组权限比用户权限大”，而是 “用户最终权限 = 用户权限 ∪ 所有组权限”。

- 换句话说：
没有优先级之分（谁大谁小），而是合并。
只要有一个地方给了 允许，最终就是允许。
只要有一个地方给了 拒绝，最终就是拒绝（拒绝优先级最高，会覆盖所有允许）。

🔎 举个例子：
usera 直接权限：读取
groupa 权限：完全控制
👉 最终：完全控制（因为“读取” + “完全控制” = “完全控制”）。

再举一个：
usera 直接权限：读取
groupa 权限：完全控制
usera 直接权限里另外勾了一个“拒绝写入”
👉 最终：能读取，但不能写入（因为“拒绝写入”覆盖了组里的允许写入）。

✅ 所以结论是：
不是“组权限比用户权限大”，而是 允许权限会合并，拒绝权限会覆盖。
