## 一、导入

同学们，今天我们要学习的是 **DHCP 服务（Dynamic Host Configuration Protocol，动态主机配置协议）**。  
在企业网络中，主机数量往往非常多，如果每台设备都手动设置 IP 地址，不仅工作量大，还容易出错。  
DHCP 的作用，就是让服务器自动为客户端分配 IP、网关、DNS 等网络参数，实现“即插即用”。

---

## 二、DHCP 的工作机制

DHCP 的工作过程主要分为四个阶段，用一个词组概括叫做 **DORA**，即：  
**Discover → Offer → Request → Acknowledge**。

我们来逐步分析这个过程：

1. **Discover（发现）**：  
    客户端开机后广播请求，询问“谁能给我一个 IP 地址？”
    
2. **Offer（提供）**：  
    DHCP 服务器收到请求后，从作用域中选出一个可用地址，发出报价。
    
3. **Request（请求）**：  
    客户端选择其中一个报价，向对应服务器发送请求包，表示“我想用你提供的这个地址”。
    
4. **Acknowledge（确认）**：  
    服务器确认并分配地址，客户端正式获得 IP。
    

这一过程是基于 **UDP 协议** 的，使用 **67/68 端口** 进行通信。

讲到这里可以给同学们演示一下 Wireshark 抓包结果，让大家直观地看到 DORA 四个阶段的报文交换。

---

## 三、DHCP 的核心作用

DHCP 的核心功能有三点：

1. **自动分配 IP 地址** —— 减少人工配置，防止地址冲突。
    
2. **集中统一管理** —— 管理员可在服务器端统一修改网关、DNS 等参数。
    
3. **地址回收与租约机制** —— 当客户端下线后，地址自动回收供他人使用，提高资源利用率。
    

企业中几乎所有 PC、打印机、无线终端都依赖 DHCP 服务。


## 四、DHCP 服务器的安装与授权

在 Windows Server 2016 中，DHCP 服务是通过角色（Role）安装的。  
安装流程如下：

- 打开“服务器管理器” → “添加角色和功能”；
    
- 勾选“DHCP 服务器”；
    
- 安装完成后系统会提示进行**授权（Authorize）**；
    

**为什么要授权？**  
因为在域环境中，只有经过授权的 DHCP 服务器才有资格分配 IP，  
以防止非法或测试服务器干扰企业网络。

授权后，在“DHCP 控制台”中，服务器图标会从灰色变为绿色，表示工作正常。

## 五、DHCP 作用域的配置

**作用域（Scope）** 就是 DHCP 分配地址的“范围定义”。  
一个作用域包含：

- 起始和结束 IP（如 192.168.xx.50–192.168.xx.100）；
    
- 子网掩码；
    
- 默认网关（Router）；
    
- DNS 服务器地址；
    
- 租约期限（默认 8 天）。
    

创建作用域后，客户端才能从中获取到合法的 IP 地址。  
可以让学生注意到：若作用域未激活（红叉），客户端将无法获得地址。

---

## 六、排除范围与保留地址

在企业网络中，有些主机（如服务器、打印机）需要使用固定 IP，  
此时我们可以通过两种方式管理：

1. **排除范围（Exclusion Range）**：  
    告诉 DHCP “这些地址不要分配”，通常用于静态设备区段。
    
2. **保留地址（Reservation）**：  
    让某个特定 MAC 地址的设备永远获得固定 IP。
    

讲解时可以举例：  
如打印机 MAC 为 `00-1A-2B-3C-xx-xx`，保留 IP 为 `192.168.xx.88`，  
这样无论重启多少次，地址都不会改变。

---

## 七、DHCP 的租约与续租机制

DHCP 分配的地址不是永久的，而是有“租约期（Lease Time）”。  
默认租期是 8 天，过半时客户端会自动续租。  
如果服务器拒绝续租，客户端会重新广播请求新的地址。

**优点：** 避免长期占用无效地址，提高地址利用率。

---

## 八、地址冲突与排查方法

在实际网络中，常见的冲突有两种：

1. **静态地址与作用域重叠**：管理员手动设置了一个 DHCP 范围内的地址；
    
2. **多个 DHCP 服务冲突**：同网段有未授权的 DHCP 服务器。
    

排查方法：

- 在客户端执行 `ipconfig /all` 查看服务器来源；
    
- 使用 `arp -a` 检查 MAC 对应关系；
    
- 若有非法服务器，可在交换机上通过 DHCP Snooping 限制来源端口。
