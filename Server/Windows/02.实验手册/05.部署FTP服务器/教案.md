# 《FTP服务的部署与用户隔离》教案  
（以 Windows Server 2016 IIS FTP 服务为教学案例）

---

### 一、课程内容
掌握 FTP 服务的安装与配置流程，理解 FTP 用户身份验证与授权机制，能够配置 FTP 用户权限隔离与目录隔离，实现对企业文件共享服务的安全分级管理。  

---

### 二、课时安排
共 2 课时（理论讲授 + 实践操作）

---

### 三、教学方式
讲授 + 实验演示 + 任务驱动 + 小组操作  

---

### 四、教学目标
#### （一）知识目标
- 理解 FTP 协议的基本原理与工作模式（主动 / 被动）；  
- 掌握在 Windows Server 2016 上部署 IIS FTP 服务的流程；  
- 熟悉 FTP 的身份验证与授权机制（匿名访问、基本身份认证）；  
- 理解并掌握用户权限隔离与目录隔离的配置方法。  

#### （二）能力目标
- 能够独立完成 FTP 服务的安装与站点创建；  
- 能在 IIS 管理器中配置 FTP 用户访问控制策略；  
- 能通过客户端（FileZilla 或资源管理器）验证不同账户的访问权限；  
- 能分析与排查 FTP 访问失败的常见原因。  

#### （三）素质目标（含思政元素）
- 通过 FTP 权限管理的学习，培养学生的安全意识与责任感；  
- 理解网络服务中“分级授权”“最小权限”的安全理念；  
- 树立信息安全“守责、守信、守法”的职业素养，弘扬工匠精神。  

---

### 五、教学重点
- FTP 服务安装、站点绑定与授权配置；  
- FTP 用户的权限隔离与目录隔离方法；  
- FTP 主动 / 被动模式的配置与防火墙端口开放。

---

### 六、教学难点
- 理解 FTP 身份验证方式与 NTFS 权限结合的机制；  
- 正确实现用户隔离目录的映射关系与权限控制。  

---

### 七、教学方法
讲授法、演示法、任务驱动法、实验验证法  

---

### 八、教学过程

#### （一）导入环节（约10分钟）
1. 教师提问：“企业中为什么要使用 FTP 服务？文件共享能否直接通过共享文件夹实现？”  
2. 引导学生思考文件传输的安全性、权限控制与远程访问需求。  
3. 说明本节课目标：从 FTP 服务基础部署 → 权限隔离 → 目录隔离，完成完整服务建设过程。  

---

#### （二）讲解示范环节（约50分钟）

##### 1. FTP 服务基础部署
- **实验环境**  
  - 两台虚拟机：  
    - iLgj-Server-xx-01：Windows Server 2016，IP：192.168.xx.10  
    - iLgj-Server-xx-02：Windows Server 2016，IP：192.168.xx.11  
- **任务步骤**  
  1. 在 iLgj-Server-xx-01 上安装 IIS FTP 服务角色；  
  2. 在 `C:\FTP1xx` 下创建站点根目录，写入测试文件 `welcome.txt` 内容：“Hello FTP xx”；  
  3. 在 IIS 管理器中创建站点：  
     - 名称：FTP1xx  
     - 物理路径：C:\FTP1xx  
     - 绑定 IP：192.168.xx.10 端口 21  
     - 启用匿名和基本身份认证  
     - 授权：所有用户读写  
  4. 创建本地用户 `ftpuserxxa`（用户名密码相同）；  
  5. 为匿名用户设置只读权限，为 `ftpuserxxa` 设置读写权限；  
  6. 在另一台虚拟机通过资源管理器访问 `ftp://192.168.xx.10` 测试匿名和授权登录。  

> **演示重点**  
> - 匿名用户与基本认证用户访问效果对比；  
> - 防火墙端口21与被动端口范围开放；  
> - 客户端上传、下载与新建文件夹验证。  

---

##### 2. FTP 用户隔离（权限隔离）
- **实验目的**  
  理解 FTP 用户隔离的意义，通过权限区分不同用户的访问能力。  
- **操作步骤**  
  1. 创建站点根目录 `C:\FTP2xx`；  
  2. 在根目录下创建子目录 `ftp2xxA`、`ftp2xxB`；  
  3. 在本地系统中创建用户 `ftp2xxA` 与 `ftp2xxB`，密码均为 123456；  
  4. 在 IIS 中创建新 FTP 站点 `FTP2xx`：  
     - 物理路径：C:\FTP2xx  
     - 身份验证：匿名 + 基本认证  
     - 授权：所有用户读写；  
  5. 配置用户隔离：  
     - `ftp2xxA` 用户读写权限；  
     - `ftp2xxB` 用户只读权限。  
  6. 在客户端分别使用两用户登录测试，验证上传 / 新建文件权限差异。  

> **讲解要点**  
> - 权限隔离体现“最小权限”原则；  
> - NTFS 权限与 IIS 授权共同决定最终权限；  
> - 可通过 FileZilla 查看 FTP 日志以验证用户根目录路径。

---

##### 3. FTP 用户隔离（目录隔离）
- **实验目的**  
  让不同用户登录后进入各自独立的目录，实现逻辑隔离。  
- **操作步骤**  
  1. 创建目录结构 `C:\FTP3xx\LocalUser\ftp3xxA`、`ftp3xxB`；  
  2. 创建用户 `ftp3xxA`、`ftp3xxB`，密码均为 123456；  
  3. 在 IIS 管理器中创建新站点 `FTP3xx`：  
     - 根目录：C:\FTP3xx  
     - 用户隔离模式：用户各自目录（User name directory, under root）  
     - 身份验证：匿名 + 基本认证；  
     - 授权规则：  
       - `ftp3xxA` → 只读  
       - `ftp3xxB` → 读写  
  4. 使用 `ftp://192.168.xx.10` 登录验证：  
     - `ftp3xxA` 登录后仅能访问其目录，无法切换至他人目录；  
     - `ftp3xxB` 可上传 / 修改文件。  

> **对比分析**  
> - 权限隔离：同目录内不同用户权限不同；  
> - 目录隔离：用户直接被锁定在各自目录；  
> - 两者结合可实现企业级 FTP 访问控制。  

---

#### （三）任务实施环节（约20分钟）
学生分组完成以下任务：  
1. 组A：部署 FTP 基础服务并测试匿名访问。  
2. 组B：配置权限隔离，分别登录验证读写权限。  
3. 组C：配置目录隔离，验证目录锁定效果。  
4. 各组提交截图与实验报告，教师巡查并讲评。  

---

#### （四）分析总结环节（约10分钟）
- FTP 的核心安全策略是身份认证与权限分级；  
- “权限隔离”控制操作行为，“目录隔离”控制访问范围；  
- 通过实验，学生应掌握 FTP 服务完整部署流程与排错思路。  

---

### 九、课后拓展
- 阅读资料：《IIS FTP 服务官方文档》《FileZilla 使用指南》；  
- 思考题：  
  - 外部客户端无法访问 FTP 时，应从哪些方面排查？  
  - 匿名访问与 Windows 身份验证的本质区别？  
  - 为什么建议将站点根目录放在非系统盘？  
- 实践延伸：  
  - 尝试配置 FTP 被动模式端口范围与防火墙规则；  
  - 在企业文件共享场景中设计基于用户组的 FTP 权限结构。  

---

### 十、教学反思
- 学生对 FTP 认证方式与 NTFS 权限关联理解不够深入，可通过示例强化；  
- 权限隔离与目录隔离实验操作性强，适合作为阶段考核内容；  
- 思政融入自然，学生能理解“安全有界限、职责有分工”的信息安全理念。
