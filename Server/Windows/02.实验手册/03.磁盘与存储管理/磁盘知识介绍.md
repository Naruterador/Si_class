| -------------- | ---- | -------- | -------------- |

| SATA III       | 串行   | 600MB/s  | HDD / SATA SSD |

#### scsi接口介绍
- SCSI（Small Computer System Interface，小型计算机系统接口）硬盘是一类通过 **SCSI 总线**进行数据传输的硬盘，主要应用于高性能服务器、工作站以及部分企业级存储设备。
- scsi特点：
    - **高可靠性**：相比 IDE/ATA 硬盘，SCSI 硬盘设计更适合 7×24 小时运行。
    - **高转速**：常见转速有 10,000 RPM、15,000 RPM，寻道速度快。
    - **支持多任务**：SCSI 控制器支持命令队列（Command Queuing），并行处理多个 I/O 请求，提升性能。
    - **常用于服务器/存储**：早期企业存储阵列（如 RAID 阵列柜）大量使用 SCSI 接口。

## MBR vs GPT 对比
| 特性     | MBR              | GPT               |
| ------ | ---------------- | ----------------- |
| 最大磁盘容量 | 2TB              | 理论上 9.4ZB（几乎无限）   |
| 最大分区数  | 4 个主分区（或扩展+逻辑分区） | 默认 128 个（可扩展）     |
| 分区表位置  | 磁盘首扇区（LBA 0）     | 磁盘首尾各一份（冗余）       |
| 启动模式支持 | BIOS             | UEFI              |
| 容错能力   | 无冗余，容易损坏         | 有主备分区表，更可靠        |
| 兼容性    | 几乎所有系统支持         | 旧系统（如 XP 32 位）不支持 |
## **常见应用场景**
- **小于 2TB 的磁盘 + 老旧 BIOS 设备**：推荐 MBR。
- **大于 2TB 的磁盘 + 现代 UEFI 主板**：必须使用 GPT。
- **服务器 / 企业环境**：通常 GPT 更合适，因为安全、容量和冗余更强。
## FAT32 和 NTFS 文件系统 对比
| 特性       | FAT32                      | NTFS                       |
| -------- | -------------------------- | -------------------------- |
| 最大单个文件大小 | 4GB − 1B                   | 理论上 16EB（Windows 限制 256TB） |
| 最大分区大小   | 理论 8TB（Windows 格式化限制 32GB） | 理论 16EB（Windows 支持到 256TB） |
| 跨平台兼容性   | 非常好，几乎所有系统支持               | 一般（Windows 默认支持最好）         |
| 安全性      | 无权限、无加密支持                  | 支持 ACL 权限、加密、压缩            |
| 容错能力     | 无日志功能，掉电易损坏                | 有日志功能，稳定性更高                |
| 常见使用场景   | U 盘、SD 卡、相机、嵌入式设备          | Windows 系统盘、大容量硬盘          |
## **应用场景**
- **FAT32**：兼容性优先 → U 盘、SD 卡、车机、相机、游戏机。
- **NTFS**：功能与安全优先 → Windows 系统盘、移动硬盘、大文件存储。
- （顺带一提：微软还推出了 **exFAT**，解决 FAT32 的 4GB 文件限制，同时保持较好的跨平台兼容性，常见于大容量 U 盘/SD 卡）。


## **UEFI** 和 **Legacy（BIOS）** 两种启动方式
## 1. **Legacy（BIOS）启动**
- **全称**：Legacy BIOS（Basic Input/Output System）
- **历史**：诞生于 1980 年代，传统的计算机启动方式。
- **启动过程**：
    1. 上电 → BIOS 自检（POST）
    2. BIOS 读取 **MBR（Master Boot Record）** 的引导信息（位于硬盘第 0 扇区，512B）
    3. MBR 再加载操作系统的引导程序（如 GRUB、Windows Bootloader）
- **分区要求**：只能用 **MBR 分区表**，最大支持 **2TB 硬盘**，最多 4 个主分区。
- **特点**：
    - 兼容性好（几乎所有老系统都支持）。
    - 启动速度慢。
    - 安全性较差（没有校验机制，容易被恶意代码劫持）。
## 2. **UEFI 启动**
- **全称**：Unified Extensible Firmware Interface（统一可扩展固件接口）
- **历史**：Intel 推动的新一代固件接口，取代传统 BIOS。
- **启动过程**：
    1. 上电 → UEFI 初始化硬件
    2. UEFI 从 **EFI System Partition (ESP)** 加载 `.efi` 文件（操作系统引导程序）
    3. 引导操作系统（Windows Boot Manager、GRUB2、Linux Shim 等）
- **分区要求**：通常搭配 **GPT 分区表**，支持超过 **2TB 硬盘**，分区数几乎无限制。
- **特点**：
    - 启动速度快。
    - 支持图形化界面和鼠标操作。
    - 支持 **安全启动（Secure Boot）**，防止恶意引导程序。
    - 可直接引导大容量硬盘（>2TB）。

- 两者对比

| 对比项   | Legacy BIOS         | UEFI                             |
| ----- | ------------------- | -------------------------------- |
| 引导方式  | MBR                 | GPT                              |
| 硬盘支持  | ≤2TB，最多 4 个主分区      | >2TB，分区数几乎无限制                    |
| 启动文件  | MBR 扇区里的引导代码        | EFI 分区里的 `.efi` 文件               |
| 启动速度  | 较慢                  | 较快                               |
| 安全性   | 无安全校验，易被篡改          | 支持 Secure Boot，更安全               |
| 界面    | 纯文本                 | 图形化，可鼠标操作                        |
| 系统兼容性 | 兼容老系统（XP、Win7 32位等） | 仅支持新系统（Win7 64位、Win10、Linux新版本等） |
## 4. **常见应用场景**
- **Legacy**：旧电脑、需要兼容 Windows XP/Win7 32 位系统的场景。
- **UEFI**：新电脑、SSD、大容量硬盘（>2TB）、需要快速启动和更高安全性的系统。
