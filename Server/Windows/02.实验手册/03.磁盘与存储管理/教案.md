### 八、教学过程

#### （一）导入环节（约10分钟）
1. 教师提问：“我们的数据都存放在硬盘上，但如果硬盘坏了怎么办？”  
2. 引导学生思考磁盘与数据安全的关系，指出企业级服务器必须具备可靠的数据冗余方案。  
3. 介绍本课内容结构：“磁盘管理基础 + RAID实践操作”。  

---

#### （二）讲解示范环节（约50分钟）

##### 1. 磁盘 / 分区 / 文件系统 三者关系
- **磁盘**：一块空地（物理硬盘/虚拟磁盘）。  
- **分区**：划地为块（空间划分）。  
- **文件系统（NTFS）**：规定如何存放与记录（相当于地上建仓库）。  
- **卷（Volume）**：格式化后的“使用单元”，拥有盘符或挂载点。  

> 比喻：磁盘是土地，分区是地块，文件系统是种植方式，卷是最终可使用的地皮。

---

##### 2. 挂载点 vs 盘符
- **盘符（C:\ D:\）**：传统、直观，最多26个字母；  
- **挂载点**：将卷挂载到现有路径下（如 `C:\Mounts\Data1`），突破盘符数量限制；  
- **场景比较**：
  - 桌面系统用盘符足够；
  - 企业服务器推荐使用挂载点（路径统一、可扩展）。  

> **演示**：  
> - 打开“磁盘管理”（`diskmgmt.msc`）；  
> - 展示如何初始化磁盘（GPT vs MBR）；  
> - 创建卷、格式化、选择盘符或挂载到空NTFS文件夹。  

---

##### 3. Windows 中的软件 RAID
**方法A：动态磁盘（教学常用）**
- 支持带区卷（RAID 0）、镜像卷（RAID 1）、RAID-5卷；  
- 步骤：
  1. 转换为动态磁盘；
  2. 创建带区/镜像/RAID-5卷；
  3. 格式化并分配盘符；
  4. 验证卷状态（健康/故障冗余）。  

**方法B：Storage Spaces（推荐生产环境）**
- 概念映射：
  - Simple → RAID 0；
  - Mirror → RAID 1；
  - Parity → RAID 5；
- GUI：Server Manager → File and Storage Services → Storage Pools → New Storage Pool。  
- 优点：灵活、支持不同容量混合、可热备、支持 PowerShell 管理。  

---

##### 4. RAID 实验演示
> 需准备：3 块以上虚拟磁盘（每块 5–10GB）

**RAID 0（带区卷）**
- 特点：性能高，无冗余；
- 步骤：创建带区卷，复制大文件观察速度；
- 风险：任意一盘坏即全部数据丢失。

**RAID 1（镜像卷）**
- 特点：冗余可靠；
- 步骤：创建镜像卷，写入文件后脱机一盘测试可读性；
- 利用率50%。

**RAID 5（奇偶校验）**
- 特点：兼顾容量与容错；
- 步骤：创建RAID-5卷或Parity虚拟盘；
- 验证断盘重建过程。

---

##### 5. RAID 对比总结表
| 项目 | RAID 0（带区） | RAID 1（镜像） | RAID 5（奇偶） |
|------|----------------|----------------|----------------|
| 最少磁盘数 | 2 | 2 | 3 |
| 可用容量 | 所有磁盘之和 | 一半 | N-1 块盘容量 |
| 读性能 | 高 | 较高 | 好 |
| 写性能 | 高 | 中 | 较低 |
| 容错能力 | 无 | 可容忍 1 块盘坏 | 可容忍 1 块盘坏 |
| 适用场景 | 临时缓存、渲染 | 系统盘、关键数据 | 文件服务器、数据库 |
