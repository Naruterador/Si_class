## 一、基础概念（通俗讲解）

### 磁盘 / 分区 / 文件系统 三者关系（比喻）

- **磁盘**：一块空地（硬盘/虚拟磁盘）。
- **分区**：把空地划成地块（分区）。
- **文件系统（NTFS）**：决定这块地怎么种菜、怎么记账（格式化为 NTFS 后才能存文件）。
- **卷（Volume）**：分区上格式化后的“使用单元”，有盘符或挂载点供系统访问。
    

### 挂载点 vs 盘符（Drive Letter）
- **盘符（C:\ D:\）**：直观、历史悠久，用户容易识别（最多 26 个字母）。
- **挂载点（空 NTFS 文件夹）**：把一个卷“挂”到已有路径下（比如 `C:\Mounts\Data1`），看起来像普通文件夹，便于扩容、无盘符限制。
- **适用场景**：
    - 普通桌面/小服务器用盘符足够。
    - 企业服务器、大容量存储常用挂载点（避免盘符耗尽、便于统一路径管理）。

## 二、实操演示：磁盘的初始化、分区、格式化与挂载（演示顺序与要点）

> 演示工具：**磁盘管理（diskmgmt.msc）** 与 **PowerShell**（推荐同时示范，GUI 对初学者友好，PowerShell 可自动化）。

### A. GUI 步骤（磁盘管理）
1. 打开“磁盘管理”（`diskmgmt.msc`）。
2. 发现新磁盘时会提示“初始化磁盘”——选择 **GPT**（现代）或 **MBR**（旧）。
3. 在未分配区域上右键 → **新建简单卷** → 指定大小 → 指定盘符或选择“挂载到空 NTFS 文件夹” → 格式化（NTFS，快速）。
4. 完成后在“此电脑”中查看新盘符或挂载点。
### B. 挂载到空文件夹（GUI）
- 在磁盘管理中，右键卷 → **更改驱动器号和路径** → **添加** → 选择“将此卷挂载到以下空 NTFS 文件夹” → 选择或新建文件夹。
### 课堂提示
- 初始化会清除磁盘上原有分区，操作前务必**确认无重要数据**。
- GPT 推荐用于新服务器和 >2TB 磁盘。

## 三、Windows 中的“软件 RAID”实现方式（两条主路线）

> Windows 上常见两种实现：**动态磁盘（Disk Management，动态卷）** 与 **Storage Spaces（存储池）**。讲师可强调：**Storage Spaces 更现代、更灵活，生产环境优先考虑**，但磁盘管理的动态磁盘方法在教学中直观易懂。

### 方法 A — 磁盘管理 + 动态磁盘（教学常用）
- 支持：**带区卷（Striped）→ RAID 0，镜像卷（Mirrored）→ RAID 1，RAID-5 卷（需 3 块）**（注：RAID-5 在部分客户端系统不可用，但在 Server 上常见）。
- 步骤（概念）：
    1. 在磁盘管理中将参与的磁盘**转换为动态磁盘**（右键磁盘 → Convert to Dynamic）。
    2. 对未分配空间右键 → New Striped Volume / New Mirrored Volume / New RAID-5 Volume。
    3. 按向导选择参与磁盘、分配盘符或挂载点、格式化为 NTFS、完成。
- 演示时注意展示：卷类型、状态（健康 / 搞错时会显示故障冗余）。
### 方法 B — Storage Spaces（推荐，现代）
- 概念映射：**Simple = 无冗余（类似 RAID 0）; Mirror = 镜像（类似 RAID 1）; Parity = 奇偶校验（类似 RAID 5）**。
- GUI 路径：Server Manager → File and Storage Services → Storage Pools → New Storage Pool（选择物理磁盘）→ New Virtual Disk（选择 Simple/Mirror/Parity）→ New Volume。
- 优点：管理灵活、支持热备、支持不同大小盘混合、PowerShell 自动化更强。
- 演示建议：用 3 块或更多虚拟磁盘演示 Parity（相当于 RAID-5）。

## 四、实操：在课堂上做三种 RAID（演示方案）

> 准备：在虚拟机上额外添加 **3 块等大小虚拟磁盘**（例如每块 5–10GB）。
### 1) 创建 RAID 0（带区卷）——示范目的：速度优先、无冗余
- GUI（磁盘管理）：
    - 全部磁盘转换为动态磁盘 → 右键未分配 → New Striped Volume → 选择两块或多块盘 → 格式化 → 完成。
- 验证：在卷上复制大文件，观察速度（感性比较）；拿单盘对比可见带区写入并行带来的速度提升。
- 风险：任何一块盘坏，数据全丢（强调：不是备份）。
### 2) 创建 RAID 1（镜像卷）——示范目的：冗余优先

- GUI：
    - 两块磁盘转换为动态 → New Mirrored Volume → 选择两块盘 → 格式化 → 完成。
- 验证：写入文件后，将一块虚拟磁盘“脱机/下线”或模拟故障，卷仍可访问（读/写仍然可读；重建后恢复冗余）。
- 注意：存储利用率 ~50%。
### 3) 创建 RAID 5（带奇偶校验）——平衡容量与容错（至少 3 块）

- GUI（Server 环境）：
    - 三块或更多磁盘动态 → New RAID-5 Volume → 选择磁盘 → 完成。
- 或使用 Storage Spaces 创建 Parity 虚拟磁盘（更推荐）。
- 验证：写入、拔掉一块磁盘（或设为脱机）观察卷为“降级/故障冗余”，再还原或替换盘重建。
- 特点：可容忍 1 块盘故障，读性能好，写有奇偶校验开销（写入性能会比 RAID 0/1 差）。

## 五、RAID 0 / RAID 1 / RAID 5 对比（课堂快速记忆表）
|项目|RAID 0（带区）|RAID 1（镜像）|RAID 5（奇偶）|
|---|---|---|---|
|最少磁盘数|2|2|3|
|可用容量|所有磁盘之和|一半（镜像）|总容量 = N-1（N块）|
|读性能|高|较高|好|
|写性能|高|中（写入到两块）|较差（奇偶校验开销）|
|容错能力|无（任一盘坏会丢数据）|可容忍 1 块盘坏|可容忍 1 块盘坏|
|适用场景|临时缓存、渲染、非关键高速场景|系统盘、关键数据（小容量）|文件服务器、数据库（容错+空间）|
