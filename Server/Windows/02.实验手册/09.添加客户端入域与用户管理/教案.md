# **《添加客户端入域与用户管理》教案**

（以 Windows Server 2016 为教学案例）

---

### 一、课程内容

本课程围绕 Windows Server 域环境中的客户端管理展开。学生将学习如何将客户端计算机加入域、如何在 Active Directory 中创建组织单位（OU）与域用户账户，并掌握组策略（GPO）的基本使用方法，包括密码策略、USB 禁用、桌面壁纸配置、文件夹重定向与软件自动安装等操作。课程旨在使学生理解域环境带来的集中管理优势。

---

### 二、课时安排

共 **2 课时**（理论讲授 + 实验操作）

---

### 三、教学方式

讲授 + 演示 + 任务驱动 + 小组实验

---

### 四、教学目标

#### （一）知识目标

- 理解客户端加入域所需的条件与基本流程；
    
- 掌握 OU、域用户、域计算机、GPO 等核心概念；
    
- 理解组策略的生效顺序与常用管理场景；
    
- 了解文档重定向、桌面壁纸强制设定、USB 禁用等配置的含义与应用价值。
    

#### （二）能力目标

- 能够在 Server 2016 中创建组织单位并创建用户；
    
- 能将客户端计算机加入域并成功使用域账户登录；
    
- 能创建并链接组策略对象（GPO），并对其进行配置；
    
- 能在客户端验证各项策略（USB、壁纸、重定向、软件安装）是否正确生效。
    

#### （三）素质目标（含思政元素）

- 培养学生统一规划、集中管理的工程思维，理解企业 IT 管理的规范性要求；
    
- 通过策略统一下发与权限控制，强化学生安全意识与责任意识；
    
- 在小组协作与实验操作中提升合作精神、严谨态度与问题分析能力。
    

---

### 五、教学重点

- 客户端加入域的流程与错误排查；
    
- 组策略的配置与生效机制；
    
- 文件夹重定向、USB 禁用、软件自动安装等策略的实际配置。
    

---

### 六、教学难点

- 理解 GPO 生效层级（本地 → 站点 → 域 → OU）以及覆盖关系；
    
- 文件夹重定向路径、共享权限与 NTFS 权限之间的关联。
    

---

### 七、教学方法

讲授法、演示法、任务驱动法、实验验证法

---

### 八、教学过程

---

#### （一）导入环节（约 10 分钟）

1. 教师提问：“如果企业有 500 台电脑，每台都要单独设置用户、桌面、密码策略，会发生什么？”
    
2. 学生回答后，教师引导总结：
    
    - 本地账户分散，管理成本高；
        
    - 无法统一安全策略；
        
    - 难以保证企业内部的安全规范与统一环境。
        
3. 教师引出本节主题：
    
    - 通过“域 + AD + GPO”可以集中管理用户与计算机；
        
    - 本节课将完成客户端入域、用户管理与策略集中下发。
        

---

#### （二）讲解示范环节（约 50 分钟）

---

##### 1. 实验拓扑与环境说明

本课程采用 VMware Workstation 虚拟化环境，使用三台虚拟机：  
（拓扑与网络配置基于实验手册内容）

|机器名|系统版本|内存|硬盘|CPU|角色|
|---|---|---|---|---|---|
|iLgj-Server-xx-01|Win2016|2G|50G|1核|域控制器（DC）|
|iLgj-Client-xx-01|Win2016|2G|50G|1核|域成员客户端|
|iLgj-Client-xx-02|Win2016|2G|50G|1核|域成员客户端|

网络要求：

- DC：192.168.xx.10
    
- Client1：192.168.xx.11
    
- Client2：192.168.xx.12
    
- DNS 均指向 DC 的 IP 地址。
    

教师强调：DNS 配置错误是客户端无法加入域的最主要原因。

---

##### 2. 将 Server 提升为新森林根域控制器

教师操作演示（学生同步练习）：

1. 打开“服务器管理器 → 添加角色和功能”；
    
2. 勾选“Active Directory 域服务（AD DS）”；
    
3. 安装完成后点击“将此服务器提升为域控制器”；
    
4. 选择“新建林”，域名输入：`contosoxx.com`；
    
5. 其余选项保持默认，设置 DSRM 密码；
    
6. 提升完成后自动重启；
    
7. 登录后打开“Active Directory 用户和计算机”验证域结构。
    

---

##### 3. 将客户端加入 contosoxx.com 域

教师强调加入域的三个条件：DNS 正确、时间同步、网络互通。

学生在两台客户端上分别执行：

1. 在“系统属性 → 计算机名”界面中选择“加入域”；
    
2. 输入域名 `contosoxx.com`；
    
3. 输入域管理员密码；
    
4. 出现“欢迎加入域”提示后重启；
    
5. 登录界面选择“其他用户”，使用 `contosoxx\Administrator` 登录验证。
    

学生需截图显示“计算机已成功加入域”。

---

##### 4. 创建组织单位与用户

在域控制器上操作（教师演示）：

1. 打开“Active Directory 用户和计算机”；
    
2. 在域根目录下新建 OU：**Deptxx**；
    
3. 在 Deptxx 中创建用户：
    
    - userxx1
        
    - userxx2
        
4. 设置密码：`P@ssw0rd!`；
    
5. 完成创建后截图 OU 和用户信息。
    

---

##### 5. 域用户登录验证

学生分别在两台客户端上：

- 使用 userxx1、userxx2 登录；
    
- 打开 CMD 输入：
    
    `whoami`
    
- 验证输出是否为：
    
    - `contosoxx\userxx1`
        
    - `contosoxx\userxx2`
        

---

##### 6. 创建 GPO_Deptxx 并配置策略

教师在“组策略管理”控制台中演示以下策略配置：

1. 新建策略：GPO_Deptxx
    
2. 链接到 OU：Deptxx
    
3. 配置以下策略：
    
    - 密码策略：最小长度 8，启用复杂性要求
        
    - 禁止 USB 存储设备访问
        
    - 文件夹重定向：将“文档”重定向至
        
        `\\DC-xx\Profiles\%username%\Documents`
        
    - 桌面壁纸统一设置为
        
        `\\DC-xx\Share\wall.jpg`
        
    - 软件安装策略：为域成员主机自动安装 python 环境
        

教师讲解 GPO 生效顺序（L → S → D → OU），并说明后者优先级更高。

---

##### 7. 策略刷新与客户端验证

学生在 iLgj-Client-xx-02 上以 userxx2 登录：

1. 运行：
    
    `gpupdate /force`
    
2. 验证以下内容是否生效：
    
    - USB 禁用
        
    - 文档位置是否已被重定向
        
    - 桌面壁纸是否更改
        
    - python 软件是否已安装
        

---

#### （三）课后总结（约 10 分钟）

教师总结以下问题内容：

1. 客户端加入域失败的常见原因：
    
    - DNS 指向错误、时间不同步、网络不通、服务异常、权限不足。
        
2. 组策略的生效顺序：
    
    - 本地 → 站点 → 域 → OU，后应用的策略覆盖前者。
        
3. 为什么企业要按部门划分 OU：
    
    - 便于分级管理、权限委派、策略分离、安全隔离、结构清晰。
