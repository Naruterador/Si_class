## 一、导入

同学们，前面我们已经认识了 Active Directory 域服务（AD DS）、理解了什么是“域”，也完成了基础的域环境搭建。  
但在实际企业环境中，仅仅“有了域”还远远不够，我们还需要：

- 让大量客户端 **加入域**，统一由域控制器管理；
    
- 通过 **组织单位（OU）** 对不同部门的用户进行分类；
    
- 使用 **组策略（GPO）** 对客户端进行集中控制，例如：
    
    - 统一密码复杂度；
        
    - 禁用 USB 存储设备；
        
    - 统一下发桌面壁纸；
        
    - 让“文档”等文件夹自动重定向到服务器；
        
    - 自动为域内主机安装指定软件（例如 python 环境）。
        

这一节课，我们将围绕一个场景开展：

> “为 Deptxx 部门统一配置安全策略和桌面环境，并让两台客户端作为域成员，完全受域控制器统一管理。”

---

## 二、域环境回顾与本节任务

在正式动手前，先快速回顾一下当前的实验环境：

- 服务器：`iLgj-Server-xx-01`
    
    - Windows Server 2016
        
    - 已固定 IP：`192.168.xx.10`
        
    - 将被提升为 contosoxx.com 的森林根域控制器
        
- 客户端：
    
    - `iLgj-Client-xx-01`：Win2016，IP 192.168.xx.11
        
    - `iLgj-Client-xx-02`：Win2016，IP 192.168.xx.12
        

> 图中 xx 为学号，两台客户端后续都将加入 `contosoxx.com` 域。

本节课的主要任务可以概括为四句话：

1. 把 `iLgj-Server-xx-01` **提升为域控制器**；
    
2. 把两台客户端 **加入域**；
    
3. 在 AD 中 **创建 OU 与用户**，让客户端使用域用户登录；
    
4. 通过 **GPO_Deptxx** 对 Deptxx OU 下的用户统一下发策略。

## 三、域控制器与 AD 结构快速回顾

教师可以用一页 PPT 复习 AD 结构层级（森林→域→OU→对象），这里不再展开长篇解释，只强调和本节实验的关系：

1. **森林（Forest）**：
    
    - 我们的实验中，整个森林只有一个域：`contosoxx.com`。
        
2. **域（Domain）**：
    
    - 安全边界，本次所有客户端、用户都在这个域里。
        
3. **组织单位（OU）**：
    
    - 我们会新建 `Deptxx`，用来容纳本次实验中要管理的用户账户，方便后面把 GPO 链接到这个 OU。
        
4. **对象（Object）**：
    
    - 本节我们主要接触的对象：用户（userxx1、userxx2）和计算机（两台客户端）。
        

> 可以简要展示上一节画过的“树形图”，加强学生记忆。

## 四、安装 AD DS 并将服务器提升为域控制器

这一部分教师以演示为主，学生同步跟做：

1. 在 `iLgj-Server-xx-01` 上打开“服务器管理器 → 添加角色和功能”；
    
2. 勾选 “Active Directory 域服务（AD DS）”；
    
3. 安装完成后，点击右上角黄色感叹号，选择“将此服务器提升为域控制器”；
    
4. 选择“新建林（Forest）”，输入域名：`contosoxx.com`；
    
5. 保持默认的林、域功能级别；
    
6. 设置 DSRM 恢复密码；
    
7. 完成后重启服务器，并使用域管理员 `contosoxx\Administrator` 登录。
    

讲解要点：

- 提醒学生：提升域控需要固定 IP，DNS 指向自己；
    
- 强调：这个服务器之后就是“整个域的核心”。

## 五、客户端加入域的操作

目标：将 `iLgj-Client-xx-01` 和 `iLgj-Client-xx-02` 都加入 `contosoxx.com` 域。

### 1. 前提检查

在两台客户端上分别确认：

- IP 地址为 `192.168.xx.11` / `192.168.xx.12`；
    
- 子网掩码 `255.255.255.0`；
    
- DNS 指向 **域控制器 IP：192.168.xx.10**；
    
- 能 ping 通 `192.168.xx.10`。
    

### 2. 加入域的步骤

以 `iLgj-Client-xx-01` 为例：

1. 打开“此电脑 → 属性 → 高级系统设置 → 计算机名”；
    
2. 点击“更改”，在“隶属于”中选择“域”；
    
3. 输入域名：`contosoxx.com`；
    
4. 输入域管理员账号与密码；
    
5. 弹出“欢迎加入 contosoxx 域”的提示后，点击确定并重启；
    
6. 重启后在登录界面选择“其他用户”，使用 `contosoxx\userxx1` 或 `contosoxx\Administrator` 登录。
    

学生按同样方法操作 `iLgj-Client-xx-02`，并按实验手册要求截图“计算机已加入域”的界面。

## 六、在 AD 中创建 OU 与用户账户

域环境搭建好、客户端也加入域之后，就可以开始做“组织与用户”的管理了。

### 1. 创建 OU：Deptxx

在 `iLgj-Server-xx-01` 上：

1. 打开“服务器管理器 → 工具 → Active Directory 用户和计算机”；
    
2. 展开左侧树，找到域名 `contosoxx.com`；
    
3. 右键域名 → “新建 → 组织单位（OU）”；
    
4. 命名为：`Deptxx`，勾选“防止意外删除此容器”。
    

### 2. 创建域用户：userxx1、userxx2

1. 在 OU `Deptxx` 下右键 → “新建 → 用户”；
    
2. 依次创建 `userxx1`、`userxx2`；
    
3. 初始密码可以统一设为：`P@ssw0rd!`；
    
4. 可选择“用户下次登录时必须更改密码”选项（可由教师要求是否勾选）。
    

学生需截图显示 Deptxx OU 和两个用户创建完成的界面。

### 3. 域用户登录验证

- 在 `iLgj-Client-xx-01` 使用 `contosoxx\userxx1` 登录；
    
- 在 `iLgj-Client-xx-02` 使用 `contosoxx\userxx2` 登录；
    
- 打开命令提示符，执行：
    
    `whoami`
    
- 看到输出为 `contosoxx\userxx1` / `contosoxx\userxx2` 即说明已使用域账户登录。

## 七、组策略（GPO）与客户端集中管理

这一部分是本节课的“核心价值点”：通过 GPO 对 Deptxx 部门的域用户统一实施策略。

### 1. 组策略的作用简述

- 集中控制客户端安全设置、桌面环境、软件安装等；
    
- 以 OU 为单位设置范围，方便按部门管理；
    
- 生效顺序：本地 → 站点 → 域 → OU（后者覆盖前者）。
    

### 2. 创建并链接 GPO_Deptxx

1. 打开“组策略管理”（GPMC）；
    
2. 找到域 `contosoxx.com` → 组织单位 `Deptxx`；
    
3. 右键 Deptxx → “在此域中创建 GPO 并在此处链接”；
    
4. 命名为：`GPO_Deptxx`。
    

---

## 八、在 GPO_Deptxx 中配置策略（与实验需求逐项对应）

根据实验手册中的要求，对 `GPO_Deptxx` 进行配置：

### 1. 密码策略

> 要求：设置最小长度 8，启用复杂性要求

- 位置（域级/默认域策略或在此示例中讲清差异）：
    
    - 计算机配置 → Windows 设置 → 安全设置 → 账户策略 → 密码策略
        

讲解要点：

- 密码复杂性要求包含大小写、数字、特殊字符；
    
- 最小长度 8 是企业常见基线之一。
    

---

### 2. 禁止 USB 存储访问

> 要求：禁止 USB 存储设备访问

- 常见配置路径示例：
    
    - 计算机配置 / 用户配置 → 管理模板 → 系统 → 可移动存储访问
        
- 可以设置“所有可移动存储类：拒绝所有访问”。
    

提醒学生：这是防止数据泄露的常用策略。

---

### 3. 文件夹重定向 “文档” 到服务器

> 要求：将“文档”重定向到 `\\DC-xx\Profiles\%username%\Documents`

- 路径：用户配置 → Windows 设置 → 文件夹重定向 → 文档
    
- 目标选择：
    
    - “基本—将每个人的文件夹重定向到同一位置”；
        
    - 路径填写为：`\\DC-xx\Profiles\%username%\Documents`。
        

讲解要点：

- 这样可以让用户数据集中存放在服务器，便于备份与迁移；
    
- 结合共享权限与 NTFS 权限设置说明安全性。
    

---

### 4. 修改桌面壁纸为统一图片

> 要求：统一桌面壁纸，例如 `\\DC-xx\Share\wall.jpg`

- 路径：用户配置 → 管理模板 → 桌面 → 桌面 → “桌面壁纸”；
    
- 指定壁纸路径：`\\DC-xx\Share\wall.jpg`；
    
- 壁纸样式可选填“填充”、“适应”等。
    

讲解要点：

- 企业通常会下发统一形象、标语等壁纸；
    
- 也可以用于“安全提示”、“保密提醒”等。
    

---

### 5. 通过软件安装策略为域成员主机安装 python 环境

> 要求：通过 GPO 软件安装为域成员安装 python 环境

- 准备：
    
    - 在 `\\DC-xx\Share` 共享目录中放置 Python 的 MSI 安装包；
        
- 路径：
    
    - 计算机配置 → 策略 → 软件设置 → 软件安装；
        
    - 新建软件包，选择网络路径 `\\DC-xx\Share\xxx.msi`；
        
    - 选择“已分配”。
        

讲解要点：

- GPO 安装软件要求使用网络路径，而不是本地盘路径；
    
- 适用于域内 PC 统一安装常用软件（浏览器、办公软件、开发环境等）。

## 九、客户端刷新策略与效果验证

按实验要求，在 `iLgj-Client-xx-02` 上以 `userxx2` 登录，完成以下操作并验证：

1. 命令行执行：
    
    `gpupdate /force`
    
2. 检查：
    
    - 修改密码时是否被要求必须复杂且长度 ≥8；
        
    - 插入 USB 存储设备是否被拒绝访问；
        
    - “文档”文件夹属性中位置是否变成 `\\DC-xx\Profiles\userxx2\Documents`；
        
    - 桌面壁纸是否已经变更为统一图片；
        
    - “程序和功能”中是否已出现通过 GPO 安装的 python 环境。

## 十、课堂演示总结

最后，回到实验手册中的三个总结问题，用来收尾整节课：

1. **客户端加入域失败的常见原因有哪些？**
    
    - DNS 未指向域控、时间不同步、网络不通、权限不足、服务异常等。
        
2. **组策略的生效顺序是怎样的？**
    
    - 本地 → 站点 → 域 → OU，后应用的优先级更高。
        
3. **在实际企业环境中，为什么要将不同部门的用户划分到不同 OU 中？**
    
    - 便于分级管理、按部门下发不同策略、结构清晰、支持权限委派。
