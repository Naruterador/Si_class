## 一、导入

同学们，今天我们要学习的是 **Active Directory 域服务（AD DS）**。  
在企业网络中，通常会有成百上千台计算机。  
如果每台电脑都单独管理用户、设置权限、安装软件，那工作量将非常庞大。  
**域（Domain）** 技术的出现，就是为了解决“集中管理”的问题。


--- 
## 二、Active Directory 域服务（AD DS）的作用与结构

**Active Directory** 是微软的目录服务，用来集中存储和管理网络中的对象信息。  
而 **AD DS（Active Directory Domain Services）** 则是它在服务器中实现的核心角色。

### 1. AD DS 的核心作用

- 统一存储域内所有用户、计算机、打印机、共享文件夹等信息；
- 提供身份验证与访问控制（用户登录、权限验证）；
- 实现集中管理（统一策略、软件分发、安全控制）；
- 支持扩展性：多个域可组成森林，实现跨部门、跨区域管理。
    

### 2. AD DS 的结构层级

从上到下依次为：

1. **森林（Forest）** —— 整个组织的最高级别结构；
2. **域（Domain）** —— 安全边界单位；
3. **组织单位 OU（Organizational Unit）** —— 用于分类和委派管理；
4. **对象（Object）** —— 用户、计算机、组、打印机等。

> 讲解时可以画出一棵“树”形图：  
> 森林 → 域 → OU → 用户 / 计算机。

## 三、工作组与域的区别

最后，我们要理解“**工作组**”与“**域**”这两个概念的区别。

| 对比项   | 工作组（Workgroup） | 域（Domain）        |
| ----- | -------------- | ---------------- |
| 管理方式  | 每台计算机独立管理      | 集中管理             |
| 用户账户  | 本地账户，互不通用      | 统一由域控制器管理        |
| 安全与权限 | 由各自电脑控制        | 统一策略控制           |
| 网络规模  | 小型局域网（10 台以内）  | 大中型网络（企业级）       |
| 登录方式  | 登录本机           | 登录域账户，可在任意域内电脑登录 |

教师可举例说明：  
在家里几台电脑互联就是工作组，而在企业中，员工使用域账户登录公司电脑，则属于域环境。

> **总结一句话：**  
> 工作组是“各管各的”，域是“集中管理”。

## 四、安装 AD DS 角色并提升为域控制器

接下来我们进入实际部署环节。  
以 **Windows Server 2016** 为例，我们需要在服务器上安装 **Active Directory 域服务** 角色，并将其提升为域控制器（Domain Controller，简称 DC）。

### 操作流程：

1. 打开“服务器管理器” → “添加角色和功能”；
    
2. 勾选“Active Directory 域服务（AD DS）”；
    
3. 安装完成后，点击提示中的“将此服务器提升为域控制器”；
    
4. 选择“新建林（Forest）”，输入域名，例如：`contosoxx.com`；
    
5. 系统会自动安装 DNS，并在重启后生效；
    
6. 重启后使用域管理员账户 `Administrator@contosoxx.com` 登录。
    

> 提醒学生注意：提升域控制器是不可逆的操作，必须保证服务器固定 IP 且 DNS 指向自身。

## 五、客户端加入域的操作

域建好后，其他计算机就可以加入该域。  
我们以 `iLgj-Client-xx-01` 为例：

1. 打开“此电脑 → 属性 → 更改设置”；
    
2. 在“隶属于”选项中选择“域”；
    
3. 输入域名 `contosoxx.com`，并输入域管理员账号与密码；
    
4. 出现“欢迎加入 contosoxx 域”的提示后，重启计算机；
    
5. 重启后选择“其他用户” → 使用 `contosoxx\\userxx1` 登录。
    

> 讲解时可提醒学生：  
> 加入域失败通常是 DNS 没指向域控，或时间不同步。

## 六、在 AD 中创建组织单位（OU）与用户账户

域建立完成后，我们需要在 AD 中组织和管理用户。

1. 打开“服务器管理器 → 工具 → Active Directory 用户和计算机”；
    
2. 在左侧树形结构中展开域名 `contosoxx.com`；
    
3. 右键域名 → “新建 → 组织单位（OU）”，命名为 `ITxx`；
    
4. 在 OU 下右键“新建 → 用户”，创建 `userxx1`、`userxx2`；
    
5. 设置初始密码（如 `P@ssw0rd!`），可选择“用户下次登录时必须更改密码”。
    

创建完成后，可在客户端用这些域账户登录系统。

> 教师可演示：  
> 登录后执行命令 `whoami`，输出为 `contosoxx\\userxx1`，说明已成功登录域账户。

## 七、课堂演示总结

**教师总结要点：**

1. 域是实现集中管理的核心；
    
2. AD DS 负责身份认证与目录存储；
    
3. 域控制器（DC）是整个域的“核心服务器”；
    
4. 组织单位 OU 是管理的基础单元，可分部门设置策略；
    
5. 客户端加入域后，登录与权限都由域统一控制。
